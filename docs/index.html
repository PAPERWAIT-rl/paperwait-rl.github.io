Overview

IAA Config is a lightweight sequential configuration language designed for deterministic parsing, runtime variable management, nested object support, and format conversion.

Install VS Code Extension View Python Package

After installing the VS Code extension, manually load the included color theme from the VS Code theme selector to enable syntax highlighting and editor styling.

Core Behavior

  • Files are parsed top to bottom.
  • Values become available immediately after parsing.
  • Expressions may reference earlier variables.
  • Only bracketed expressions are evaluated.
  • Dynamic variables may be updated and written back to disk.
  • Nested variable paths using dot notation are converted into structured objects.

Rules

The parser follows a strict sequential execution model intended to keep evaluation deterministic and predictable.

Parser Rules

  • Read top to bottom.
  • Store values immediately.
  • Evaluate only bracketed expressions.
  • Later values cannot be used before they appear in the file.
  • Dynamic variables can be modified and written back.
  • Nested paths automatically create objects for each section in the path.

Dynamic variables are mutable runtime values that can be modified and persisted back into configuration files.

Python Integration

from iaacompiler import IAAModule

    "build_number",
    43,

Persistence Workflow

Dynamic values maintain their original file structure while updating only the associated value line inside the configuration document.

Nested Variables

IAA now supports nested variables using dot notation, so deeply structured configuration can be written directly in a flat file.

Dot Notation

Use . to separate path segments. Unlimited nesting depth is supported, and each section automatically becomes an object.

server.host:
 localhost

server.port:
 8080

server.settings.debug:
 True

Structured Output

{
  "server": {
    "host": "localhost",
    "port": 8080,
    "settings": {
      "debug": true
    }
  }
}

Nested Example

a.b.c.value:
 example

This becomes {"a":{"b":{"c":{"value":"example"}}}} during parsing and API conversion.

API

The IAA Conversion API converts IAA files into JSON, YAML, or TOML, and can also convert JSON, YAML, or TOML back into IAA.

Base URL
https://iaa.iaa-api.workers.dev/

Request Format

{
  "format": "json",
  "data": "server.host:\n localhost"
}

Response Example

{
  "server": {
    "host": "localhost"
  }
}
